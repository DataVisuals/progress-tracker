üêõ CRITICAL BUG DISCOVERED & FIXED

Issue: Memory Leak in Period Generation (backend/src/db.js)

Severity: Critical - Caused complete server crashes

Description: When creating a metric with an invalid frequency value (anything other than 'weekly', 'monthly', 'quarterly'), the period generation function entered an infinite loop. The while loop continued indefinitely without incrementing the date, creating unlimited periods until the Node.js process exhausted available memory (4GB heap) and crashed with fatal error.

‚úÖ Fix Applied

Added validation and safety checks in backend/src/db.js:

// Validate frequency before generating periods
const validFrequencies = ['weekly', 'monthly', 'quarterly'];
if (!validFrequencies.includes(frequency)) {
  throw new Error(`Invalid frequency: ${frequency}. Must be one of: ${validFrequencies.join(', ')}`);
}

// Safety check to prevent infinite loops
if (periods.length > 1000) {
  throw new Error('Too many periods to generate. Please check your date range and frequency.');
}

Result: Invalid frequencies now properly rejected with clear error messages, preventing server crashes
